```
Rules for this session:
- Think before acting. Read existing files before writing code.
- Be concise in output but thorough in reasoning.
- Prefer editing over rewriting whole files.
- Do not re-read files already read unless file may have changed.
- Test your code before declaring done.
- No sycophantic openers or closing fluff.
- No emojis or em-dashes.
- Keep solutions simple and direct.
- Do not guess APIs, versions, flags, commit SHAs, or package names.
- For UI changes: follow `STITCH-UI.md` and the Modern SaaS Design System tokens.
- User instructions always override this list.
```

The CLAUDE.md / GEMINI.md files in the project root already enforce these rules automatically. Use this version only when working outside the repo (web chat, ad-hoc API call, etc.).
